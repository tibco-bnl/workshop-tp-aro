# Release Notes - v1.15.0

**Release Date:** March 10, 2026  
**Last Updated:** March 10, 2026  
**TIBCO Platform Control Plane Version:** 1.15.0

## Overview

This release contains documentation and scripts aligned with **TIBCO Platform Control Plane version 1.15.0** for deploying TIBCO Platform on Azure Red Hat OpenShift (ARO) clusters.

## What's New in 1.15.0

> **Source:** [Official TIBCO Platform 1.15.0 Release Notes](https://docs.tibco.com/pub/platform-cp/latest/doc/html/Default.htm#Release-Notes/new-features.htm)

### TIBCO Control Plane New Features

#### Connectivity and Networking
- **Optional Hybrid Connectivity**: You can now enable or disable hybrid connectivity. By default, Control Plane uses hybrid connectivity to connect to data planes, but you have the option to disable it if your data plane is directly reachable from the Control Plane
- **Gateway API Controller Support**: Full support for Kubernetes Gateway API Controllers, standardizing traffic routing in Kubernetes clusters
- **Simplified DNS Naming Structure**: Control Plane now owns the first level subdomain (e.g., `controlplane.company.com`) instead of three subdomain levels. Configure the Platform Console host prefix using the `adminHostPrefix` property (default: admin)
- **OpenShift Router Support**: OpenShift Router is now supported as an Ingress Controller for Control Tower data plane - critical for ARO deployments!

#### User Interface and Experience
- **Multiple Observability Dashboards**: Add and switch between multiple observability dashboards for enhanced monitoring flexibility
- **Enhanced Header UI**: Improved header across all Control Plane UI pages for better navigation and access to contextual information
- **New Data Plane Views**: Two display options on the Data Planes page - new stacked view and existing grid view, with added filter and search functionality for managing large numbers of data planes

#### Automation and Integration
- **Platform API End-to-End Automation**: New APIs for creating subscriptions, configuring external IDPs, and OAuth configuration. Includes getting initial tokens with specific validity times and granting all permissions to initial control plane owners - enabling full environment setup without manual intervention
- **Platform CLI Custom Developer Hub URL**: Specify custom developer hub URLs to facilitate end-to-end migration and entity movement to local environments

### TIBCO BusinessWorks™ 6 (Containers)

- **Customizable Capability Recipes**: Customize BW6 (Containers) capability recipes during provisioning or update configuration (note: customizations are not preserved during upgrades)
- **Expanded Ingress Controller Support**: Support for Gateway API, Nginx, Istio, GKE, and HAProxy ingress controllers
  - Provision BW6 capabilities using these controllers
  - Access public API documentation
  - Expose application endpoints publicly
  - **OpenShift Router support** for ARO deployments

### TIBCO ActiveMatrix BusinessWorks™ 5 and Adapters

- **Events Configuration**: Configure Events for Machines and Service Instances
- **Hawk RuleBase Upload**: Upload Hawk RuleBase files for Machines and Service Instances
- **Process Starter Control**: Enable/disable Process Starters for BW5 applications via Control Plane
- **Active Alerts Visibility**: View Active Alerts for Process Archives and Machines

### TIBCO BusinessWorks™ 5 (Containers)

- **Customizable Capability Recipes**: Customize BW5 (Containers) capability recipes during provisioning or update configuration
- **Expanded Ingress Controller Support**: Support for Gateway API, Nginx, Istio, GKE, and HAProxy ingress controllers for BW5 capabilities and applications

### TIBCO Flogo Capability

- **Gateway API Controller Support**: Support for Nginx Kubernetes API Gateway, GKE Gateway, and Istio Gateway for Flogo provisioner and applications
- **HAProxy Ingress Controller**: Full support for HAProxy Ingress Controller
- **OpenShift Router Support**: Native OpenShift Router integration

### Component Version Updates

- **Developer Hub**: Version 1.15.14
- **Observability Service (o11y-service)**: Version 1.15.19 with improved monitoring
- **Event Processing Addon**: Version 1.15.0 for event-driven architectures
- **BWCE Provisioner**: Version 1.15.6
- **Flogo Provisioner**: Version 1.15.5
- **Messaging (EMS/Gateway)**: Version 1.15.31
- **Artifact Manager**: Version 1.15.2

### Deployment and Infrastructure Changes (ARO-specific)

#### Helm Chart Updates
- **Unified Chart Deployment**: Simplified deployment using `tibco-cp-base` chart version 1.15.0
- **Helm 3.13+ Required**: New label-based deployment tracking (`--labels layer=number`)
- **Data Plane Infrastructure**: New `dp-core-infrastructure` chart version 1.15.8

#### Security Enhancements
- **Enhanced Secret Management**: New **required** secrets for session keys and encryption:
  - `session-keys` secret (SESSION_KEY and SESSION_IV)
  - `cporch-encryption-secret` (CPORCH_ENCRYPTION_KEY)
- **Improved Network Policies**: Better namespace labeling and network isolation with `platform.tibco.com` labels
- **Certificate Management**: Separate certificates for `my` and `tunnel` domains recommended
- **OpenShift Security Context Constraints (SCC)**: Enhanced SCC support for TIBCO Platform components

#### Storage and Ingress
- **OpenShift Router Support**: Native OpenShift Router for ingress traffic management
- **Kong Ingress Support**: Optional Kong ingress controller for user applications on Data Plane
- **Enhanced Storage Configuration**: Improved Azure Files and Disk storage class options for OpenShift
- **dp-config-aro Chart**: ARO-specific configuration for ingress and storage setup

### Breaking Changes

⚠️ **Important:** Review these breaking changes before upgrading:

- **Helm 3.13+ Required**: Minimum Helm version increased to 3.13.0 for label support
- **Mandatory Secrets**: Session keys and encryption secrets are now required (were optional in 1.14.0)
- **Network Policy Changes**: Enhanced namespace labeling requirements with new label structure
- **Certificate Structure**: Separate certificates for `my` and `tunnel` domains now recommended
- **DNS Naming Convention**: Simplified from three-level to one-level subdomain ownership
- **Environment Variables**: New naming conventions with `TP_` and `CP_` prefixes
- **OpenShift Router Configuration**: Different ingress configuration compared to standard Kubernetes ingress controllers

## Upgrade Path

### Upgrading from v1.14.0 to v1.15.0

The TIBCO Platform team provides an automated upgrade script for upgrading from 1.14.0 to 1.15.0.

**Upgrade Script Location**: 
```bash
# Clone or pull latest tp-helm-charts repository
git clone https://github.com/TIBCOSoftware/tp-helm-charts.git
# or
cd tp-helm-charts && git pull

# Navigate to upgrade scripts
cd scripts/1.15.0

# Run interactive upgrade
./upgrade.sh
```

**Upgrade Documentation**: 
- [TIBCO Control Plane Upgrade Assistant (1.14.0 → 1.15.0)](https://github.com/TIBCOSoftware/tp-helm-charts/blob/main/scripts/1.15.0/README.md)
- [Official Upgrade Documentation](https://docs.tibco.com/pub/platform-cp/latest/doc/html/Default.htm#Installation/upgrading-tibco-control-plane.htm)
- [OpenShift-Specific Considerations](https://docs.tibco.com/pub/platform-cp/latest/doc/html/Default.htm#Installation/openshift-upgrade.htm)

### Important Upgrade Notes
- Review the [breaking changes](#breaking-changes) section above
- Backup your current values files before upgrading
- Test upgrade in non-production environment first
- Plan maintenance window for production upgrades
- Verify OpenShift SCC configurations after upgrade
- Confirm OpenShift Router configuration is compatible

## System Requirements

### Minimum ARO Cluster Specifications

#### Control Plane Cluster
- **OpenShift Version**: 4.14+
- **Node Count**: 3+ worker nodes (plus 3 master nodes managed by Azure)
- **Worker Node Size**: Standard_D8s_v3 or higher (8 vCPUs, 32GB RAM)
- **Total Worker Resources**: 24+ CPU cores, 96+ GB RAM
- **Storage**: Azure Disk (Premium_LRS) + Azure Files
- **Virtual Network**: Minimum /22 CIDR for cluster networking

#### Data Plane Cluster
- **OpenShift Version**: 4.14+
- **Node Count**: 2+ worker nodes (plus 3 master nodes managed by Azure)
- **Worker Node Size**: Standard_D4s_v3 or higher (4 vCPUs, 16GB RAM)
- **Total Worker Resources**: Based on workload (minimum 8 cores, 32GB RAM)
- **Storage**: Azure Files + Azure Disk
- **Virtual Network**: Adjacent subnet for connectivity

### OpenShift-Specific Requirements
- **Red Hat Pull Secret**: Required for ARO cluster creation
- **OpenShift CLI (oc)**: Version 4.14+ for cluster management
- **Security Context Constraints**: Proper SCC permissions configured
- **OpenShift Router**: Configured for ingress traffic
- **Service Mesh (Optional)**: Red Hat OpenShift Service Mesh for advanced routing
